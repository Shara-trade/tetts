# Обновление админ-панели v2.1

## Дата обновления
2024

## Что добавлено

### 1. Снятие роли администратора
- **Функция:** `admin_remove_role_start`, `admin_remove_role_confirm`, `admin_remove_role_execute`
- **Описание:** Creator может снимать роли администраторов и модераторов
- **Особенности:**
  - Нельзя снять роль самому себе
  - Требуется подтверждение перед снятием
  - Все действия логируются
- **Доступ:** Только Creator

### 2. Редактирование растений
- **Функции:**
  - `admin_plant_edit` - меню редактирования
  - `admin_plant_edit_field` - выбор поля для редактирования
  - `admin_plant_edit_text_value` - редактирование текстовых полей (название, эмодзи)
  - `admin_plant_edit_number_value` - редактирование числовых полей (цены, время, уровень)
- **Параметры для редактирования:**
  - Название (item_name)
  - Эмодзи (item_icon)
  - Время роста (growth_time)
  - Цена семян (buy_price)
  - Цена продажи (sell_price)
  - Урожайность (yield_amount)
  - Требуемый уровень (required_level)
  - Опыт за сбор (exp_reward)
- **Доступ:** Admin, Creator

### 3. Активация/деактивация растений
- **Функция:** `admin_plant_toggle`
- **Описание:** Переключение статуса активности растения (is_active)
- **Доступ:** Admin, Creator

### 4. Статистика использования растения
- **Функция:** `admin_plant_stat`
- **Описание:** Отображение статистики использования конкретного растения
- **Показывает:**
  - Параметры растения (цены, время роста, прибыль)
  - Количество посадок (заглушка ~0)
  - Общий заработок (заглушка ~0🪙)
- **Примечание:** Статистика в разработке
- **Доступ:** Admin, Creator

### 5. Список промокодов с пагинацией
- **Функция:** `admin_list_promo`
- **Описание:** Просмотр всех промокодов с разбивкой на страницы (5 на страницу)
- **Отображает:**
  - Код промокода
  - Тип награды
  - Использовано/Лимит
  - Срок действия
  - Статус активности
- **Кнопки действий:**
  - Редактирование (заглушка)
  - Активация/деактивация (заглушка)
  - Удаление (заглушка)
- **Навигация:** ◀️ Страница X/Y ▶️
- **Доступ:** Admin, Creator

## Новые FSM состояния

Добавлены в класс `AdminFSM`:
- `plant_edit_text` - редактирование текстовых полей растения
- `plant_edit_number` - редактирование числовых полей растения

## Изменения в заглушках

Обновлен список заглушек в функции `admin_not_implemented`:
- Удален `admin_remove_role` (реализован)
- Удален `admin_list_promo_0` (реализован)
- Добавлены `promo_edit_`, `promo_toggle_`, `promo_delete_`, `noop` (новые заглушки)

## Обновленные callback_data

### Управление админами
- `admin_remove_role` - открыть список админов для снятия роли
- `remove_admin_{user_id}` - выбрать админа для снятия
- `confirm_remove_{user_id}` - подтвердить снятие роли

### Растения
- `plant_edit_{plant_id}` - открыть меню редактирования
- `edit_plant_{field}` - выбрать поле для редактирования
- `plant_toggle_{plant_id}` - переключить активность
- `plant_stat_{plant_id}` - показать статистику

### Промокоды
- `admin_list_promo_{page}` - список промокодов с пагинацией
- `promo_edit_{code}` - редактировать промокод (заглушка)
- `promo_toggle_{code}` - переключить активность (заглушка)
- `promo_delete_{code}` - удалить промокод (заглушка)
- `noop` - пустой обработчик для кнопок без действия

## Интеграция с существующим кодом

Все новые функции интегрированы с существующими:
- Используют `get_db()` для доступа к БД
- Используют `log_admin_action()` для логирования
- Используют `check_admin_access()` для проверки прав
- Следуют паттерну FSM для пошаговых операций
- Используют `InlineKeyboardMarkup` для навигации

## Безопасность

- Снятие роли невозможно для самого себя
- Проверка прав доступа перед выполнением операций
- Логирование всех действий админов
- Подтверждение критических операций

## Требования к БД

Для работы новых функций требуются следующие таблицы:
- `admin_roles` - для управления ролями (вместо admin_users)
- `shop_config` - для редактирования растений
- `promocodes` - для списка промокодов
- `admin_logs` - для логирования действий

Все эти таблицы уже определены в `data/init_db.sql` и `data/database_schema_full.sql`
