# Что можно доработать в существующем коде

Анализ существующего кода игры «Ленивый Фермер»

---

## 1. handlers.py - основные обработчики игрока

### 1.1 Функция `show_farm()`
**Что доработать:**
- Нет проверки на `None` для `user` перед использованием полей
- `city_name` формируется некорректно (проверка `== 0` вместо `== 1` для Деревни)
- Нет обработки ошибки если `plots` пустой
- `bonus['message']` может отсутствовать в словаре (используется `.get()` но не везде)
- Эмодзи растений (`plot.get('icon')`) берётся не из всех источников

### 1.2 Функция `plant_crop()`
**Что доработать:**
- Нет проверки что `buttons` не пустой перед созданием клавиатуры
- `buttons[2:]` может быть пустым - создаётся пустая строка в клавиатуре
- Нет проверки уровня игрока перед покупкой семян
- Нет проверки что семена доступны для покупки (возможно заблокированы)
- Нет проверки что грядка действительно пустая перед посадкой

### 1.3 Функция `buy_plant()`
**Что доработать:**
- Нет обработки случая когда `crop` не найден (есть проверка но без возврата)
- `update_balance` вызывается с отрицательным числом - нет проверки достаточности баланса внутри функции
- Нет транзакционности: баланс списывается даже если посадка не удалась
- Нет проверки что семена добавляются в инвентарь перед списанием денег
- Нет логирования операции посадки

### 1.4 Функция `harvest_all()`
**Что доработать:**
- `ready_crops` собирается до `harvest_plots()` но используется после - если грядки собрались во время выполнения, будет несоответствие
- `total_earned` обновляется но `total_harvested` только через `+len(rows)`
- Нет проверки что `event['multiplier']` > 0
- `await db.update_quest_progress()` вызывается для каждой культуры отдельно - можно объединить
- Нет проверки что `ready_count` совпадает с фактически собранным

### 1.5 Функция `claim_daily()`
**Что доработать:**
- Нет проверки что `bonus['streak']` существует (используется `.get()` но потом прямое обращение)
- `await db.get_daily_bonus()` вызывается дважды - можно кэшировать
- Нет проверки что награда действительно выдана

### 1.6 Функция `shop_handler()`
**Что доработать:**
- `event['end_date']` может быть строкой, а не datetime - нет проверки типа
- Нет проверки что `event` не None перед доступом к полям
- `hours_left` может быть отрицательным если событие закончилось

### 1.7 Функция `shop_category()`
**Что доработать:**
- `seasonal_items` получаются из `fetchall()` но формат строк (tuple) а не dict
- `sell_price` с множителем события не сохраняется - цена меняется только в отображении
- Нет проверки что `category` валиден
- Нет обработки пустой категории

### 1.8 Функция `inventory_handler()`
**Что доработать:**
- Нет получения информации о растениях (только коды и количества)
- Нет разделения по категориям (семена/урожай)
- Нет возможности продать из инвентаря
- Нет отображения общей стоимости инвентаря

### 1.9 Функция `prestige_handler()`
**Что доработать:**
- Нет информации сколько осталось до следующего уровня престижа
- Нет кнопки для повышения престижа если условия выполнены
- Нет отображения бонусов следующего уровня

### 1.10 Функция `promo_handler()`
**Что доработать:**
- Нет проверки формата промокода (длина, символы)
- Нет кнопок для просмотра активных промокодов
- `result['rewards']` может не содержать ожидаемые ключи

### 1.11 Функция `top_handler()`
**Что доработать:**
- `user_rank` может быть None - нет проверки
- Нет пагинации для большого количества игроков
- Нет разных типов топов (по престижу, по урожаю)

### 1.12 Функция `fertilize_plot()`
**Что доработать:**
- Удобрение ускоряет на фиксированные 30 секунд - не масштабируется
- Нет проверки что после применения грядка всё ещё "растёт"
- Нет отображения нового времени созревания
- SQL запрос не обновляет `fertilized` флаг

### 1.13 Квесты (`quests_handler`, `claim_quest_reward`)
**Что доработать:**
- `claim_quest_reward` принимает `callback` но ожидает `message` для вызова `quests_handler`
- Нет проверки что квест действительно выполнен перед выдачей награды
- Нет уведомления если квест выполнен но награда не получена

### 1.14 Достижения (много функций)
**Что доработать:**
- Много дублирования кода для формирования сообщений о наградах
- `achievements_all()` обрезана - неполная реализация пагинации
- Нет проверки что `ach['requirement_count']` > 0 перед делением
- `is_locked` проверяется во многих местах - можно вынести в отдельную функцию
- Нет кэширования результатов `get_achievement_categories()`

## 2. keyboards.py - клавиатуры

### 2.1 `get_main_keyboard()`
**Что доработать:**
- Клавиатура статичная - нет адаптации под уровень игрока
- Нет кнопки помощи
- Нет кнопки промокодов

### 2.2 `get_farm_keyboard()`

- `fertilize_buttons` ограничен первыми 2 грядками - нет объяснения почему
- Нет кнопки для полива (если будет добавлено)
- Нет кнопки информации о грядке

### 2.3 `get_shop_categories()`
**Что доработать:**
- Статичный список - не берётся из БД
- Нет отображения доступных категорий
- Нет отметки категорий с сезонными товарами

### 2.4 `get_back_keyboard()`
**Что доработать:**
- Используется только "Назад к грядкам" - нет универсального варианта

## 3. database.py - работа с БД

### 3.1 Метод `get_user()`
**Что доработать:**
- Возвращает только часть полей из `users` - нет гибкости
- Нет параметра для выбора какие поля нужны
- `is_banned` проверка есть в SQL но не возвращается в результате

### 3.2 Метод `update_balance()`
**Что доработать:**
- Нет проверки что `amount` не приведёт к отрицательному балансу
- Нет логирования транзакции
- `transaction` параметр не используется (всегда используется транзакция)
- Нет возврата нового баланса если транзакция успешна

### 3.3 Метод `get_plots()`
**Что доработать:**
- Вызывает `_update_ready_plots()` но не проверяет результат
- `remaining` может быть отрицательным если время вышло но статус не обновился
- Нет информации о качестве урожая
- `icon` не получается из таблицы растений

### 3.4 Метод `harvest_plots()`
**Что доработать:**
- Нет возврата информации о собранных культурах
- `multiplier` берётся из `user` но не проверяется что он есть
- Нет записи в транзакции
- Нет обновления инвентаря (урожай сразу продаётся)

### 3.5 Метод `claim_daily_bonus()`
**Что доработать:**
- Нет проверки что `reward` найден в БД
- Нет возврата информации о полученных предметах
- `items` добавляются в инвентарь но нет проверки успеха

### 3.6 Метод `activate_promo()`
**Что доработать:**
- Нет проверки что `rewards` валидный JSON
- Нет проверки что награды действительно выданы
- Нет логирования активации

### 3.7 Система ачивок
**Что доработать:**
- `check_and_update_achievements()` - нет пакетной обработки (каждая ачивка отдельный запрос)
- `_unlock_next_level()` - нет проверки что следующий уровень существует
- `claim_achievement_reward()` - нет транзакционности при выдаче наград
- `get_achievement_stats()` - `is_secret` проверка только для total, но не для completed
- Много SQL запросов в циклах - можно оптимизировать

### 3.8 Метод `fertilize_plot()`
**Что доработать:**
- Ускорение на 30 секунд захардкожено
- Нет проверки что удобрение ещё не применено
- Нет обновления флага `fertilized`

## 5. admin_panel_full.py - админ-панель

### 5.1 `show_admin_menu()`
**Что доработать:**
- `edit` параметр не используется в логике - можно убрать
- Нет кэширования роли админа

### 5.2 Поиск игроков
**Что доработать:**
- Нет валидации что ID положительный
- Нет обработки случая когда username пустой
- Нет пагинации для списков

### 5.3 `show_user_profile()`
**Что доработать:**
- Нет обработки ошибки если `user_info` пустой
- `datetime.fromisoformat()` может выбросить исключение - нет try/except
- `delta` может быть отрицательным - нет проверки
- Нет отображения кристаллов
- Нет отображения опыта

### 5.4 Выдача ресурсов
**Что доработать:**
- Нет проверки что сумма не слишком большая
- Нет лимита на частоту выдачи

### 5.5 Бан система
**Что доработать:**
- Нет проверки что модератор не банит выше по роли
- `ban_expires` не проверяется на валидность
- Нет автоматической разблокировки по таймеру

### 5.6 Создание промо
**Что доработать:**
- Нет валидации дат (начало > конец возможно)
- Нет проверки что `reward_value` разумный
- `items` в награде не выбирается из списка

### 5.7 Рассылка
**Что доработать:**
- Нет обработки фото/видео (только текст)
- Нет прогресс-бара при отправке
- Нет возможности остановить рассылку
- Нет сохранения ID отправленных сообщений

### 5.8 Управление админами
**Что доработать:**
- Нет проверки что нельзя снять creator
- Нет уведомления новому админу
- Нет истории изменений ролей

**Что доработать:**

## 7. Общие проблемы

### 7.1 Обработка ошибок
- Много `try/except: pass` - ошибки игнорируются
- Нет централизованной обработки исключений
- Нет логирования ошибок

### 7.2 Производительность
- Много SQL запросов в циклах
- Нет кэширования часто используемых данных
- Нет пагинации для больших списков

### 7.3 Безопасность
- Нет rate limiting
- Нет проверки типов входных данных
- Нет защиты от SQL инъекций (хотя используются параметризованные запросы)

### 7.4 UX/UI
- Нет подтверждения для важных действий
- Нет индикаторов загрузки
- Нет обработки одновременных нажатий

### 7.5 Код
- Дублирование кода формирования сообщений о наградах
- Жёстко закодированные строки и числа
- Нет комментариев к сложным частям
- Не все функции имеют docstrings
